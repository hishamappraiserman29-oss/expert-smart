## Purpose
Evolve the platform into a "Sovereign Real Estate Intelligence Platform" using strict procedural routes. Connects frontend intent (Asset/Purpose) directly to mathematical factors and physical sheet pruning.

## Context
- **Target File**: `core_engine/sovereign_router.py`, Word/Excel report generators.
- **Trigger**: Called automatically upon clicking 'Generate' based on the frontend `val-purpose`.

## Execution Rules
1. **Mathematical Modifiers**:
   - `bank` (التمويل البنكي): Apply safety factor `x 0.95`. Produce simplified report without DCF.
   - `judicial` (التصفية القضائية): Apply forced sale factor `x 0.82`.
   - `ma` (الاستحواذ): Trigger complex Discounted Cash Flow (DCF).
2. **Pruning Engine (التنظيف الفيزيائي)**: Purge any Excel sheets or Word pages that do not match the asset type (e.g., delete farm sheets when valuing an office). Strictly cap reports to `20 pages` ideally to ensure leanness and focus.
3. **Data Independence (الضوابط السيادية)**: 
   - Never alter the baseline UTF-8 encoding or core system fonts.
   - Ensure all maps remain Static (No external dynamic API keys needed, ensuring standalone execution capability).
   - Enforce `arabic_reshaper` + `bidi.algorithm` whenever text is burned onto static maps or plots.
   - Outputs must finalize as professional `.xlsm` formats ready for signature execution.
