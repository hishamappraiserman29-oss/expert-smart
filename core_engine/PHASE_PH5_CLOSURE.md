## Module Details

### AppConfig
22 configuration fields across 7 categories (server, security, database, cache, rate-limiting,
MCP bridge, logging). All field names map to `EXPERT_SMART_<FIELD>` environment variables.
`to_dict()` redacts `secret_key`. `is_production` property: `debug=False AND secret_key set`.

**Key env vars for production:**
```bash
EXPERT_SMART_SECRET_KEY=<random-256-bit-hex>
EXPERT_SMART_DEBUG=false
EXPERT_SMART_PORT=5000
EXPERT_SMART_WORKERS=4
EXPERT_SMART_LOG_LEVEL=INFO
EXPERT_SMART_CACHE_TTL=300
EXPERT_SMART_RATE_LIMIT=60
```

### HealthChecker
- `register(name)` decorator or `register_fn(name, fn)` for direct registration
- Each check runs in a daemon thread with a configurable timeout (default 5s)
- Aggregation: UNHEALTHY > DEGRADED > HEALTHY (highest severity wins)
- `run_all()` returns `{status, checks, summary, elapsed_ms}` — drop-in for `/api/advisor/health`
- Built-in factories: `disk_space_check()`, `directory_writable_check()`, `import_check()`

### StartupValidator
- Fluent API: `v.add_python_version_check().add_directory_check(...).add_import_check(...)`
- FAIL checks → `ok_to_start = False`; WARN checks → logged only
- All checks always run (fail-all, not fail-fast) so you see every issue at startup
- `default_validator(root_dir)` creates a pre-configured validator for Expert Smart

### API_DOCS.md
- 84 routes auto-extracted from bridge_api.py via AST parsing
- Includes method, handler function name, and first-line docstring where present
- Re-generate any time with: `python PH_5_docs_deploy.py --api-docs`
