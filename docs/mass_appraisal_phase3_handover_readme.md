## 2. القيمة التجارية

**قبل Phase 3:** كانت المنصة تُنتج قيم تقييم جماعي لكل صف بشكل مستقل، بدون دعم للمناطق السوقية، ولا بيانات المبيعات، ولا أدوات المعايرة، ولا أطر الحوكمة.

**بعد Phase 3:** أصبحت المنصة قادرةً على:

- **تجميع العقارات** حسب مناطق السوق وفئات الملكية، وإنتاج ملخصات إحصائية لكل شريحة.
- **استيعاب بيانات المبيعات والتحقق منها** وفق معايير السوق المعتمدة.
- **ضبط الأسعار زمنياً** لمراعاة اتجاهات السوق.
- **تطبيق عوامل التعديل البيعي** للأخذ بعين الاعتبار الفروق في الموقع والحالة والجودة والتمويل.
- **إنتاج مؤشرات Ratio Study** وفق معايير IAAO: Median Ratio، COD، PRD.
- **معاينة اقتراحات المعايرة** دون تطبيق تلقائي، مع توصيات استرشادية.
- **اختبار سيناريوهات المعايرة** في بيئة Sandbox آمنة دون المساس بالقيم الأصلية.
- **توثيق قرارات الحوكمة** مع سجل تدقيق كامل لكل تغيير حالة.
- **إدارة بيانات إصدار النموذج** ودورة إعادة التقييم لأغراض التتبع والمراجعة.
- **تصدير المخرجات** بصيغ CSV وXLSX متعددة الأوراق وJSON للجلسة.

## 3. سير العمل الحالي

```
1. المحلل يُدخل بيانات العقارات → يضغط تشغيل → يحصل على market_value لكل صف
2. المحلل يُدخل بيانات المبيعات → يتحقق منها → يضبطها زمنياً → يُطبّق عوامل التعديل
3. يُشغّل Ratio Study → يُراجع COD/PRD/Median Ratio
4. يُشغّل Calibration Preview → يطّلع على suggested_factor
5. يُشغّل Calibration Sandbox → يرى التأثير المتوقع دون تعديل القيم الأصلية
6. يُوثّق قرار الحوكمة: مراجع/حالة/ملاحظة/مخرجات معتمدة
7. يُدخل بيانات دورة إعادة التقييم: cycle_id/model_version/تواريخ
8. يُصدّر التقرير: CSV + XLSX + جلسة JSON
```

## 4. المخرجات الرئيسية

| المخرج | المحتوى |
|--------|---------|
| **جدول نتائج التشغيل** | `market_value` + جودة البيانات + مستوى الشذوذ لكل صف |
| **ملخص المناطق** | إحصائيات لكل `zone_id`: عدد، إجمالي، متوسط، وسيط، قيمة/م² |
| **ملخص فئات العقارات** | نفسه حسب `property_class` |
| **تقرير Ratio Study** | Median Ratio، COD، PRD — على مستوى المحفظة والمنطقة والفئة |
| **معاينة المعايرة** | suggested_factor + recommendation لكل مستوى تجميع |
| **نتائج Calibration Sandbox** | sandbox_calibrated_value + delta لكل صف — القيم الأصلية محفوظة |
| **سجل الحوكمة** | governance_id + حالة + مراجع + تاريخ اعتماد + سجل تدقيق |
| **بيانات دورة التقييم** | cycle_id + model_version + تواريخ + حالة النموذج |
| **CSV** | تقرير كامل مع بيانات الحوكمة والنموذج والتدقيق |
| **XLSX** | مصنّف Excel احترافي (17+ ورقة) |

## 5. الحوكمة وقابلية التدقيق

- **سجل تدقيق كامل:** كل تغيير في حالة الحوكمة (من → إلى، بواسطة، بملاحظة، بتاريخ ووقت دقيقين) مُسجَّل في `governance_history`.
- **تتبع إصدار النموذج:** `cycle_id` و`model_version` و`model_status` تسمح بربط كل تقرير بدورة إعادة تقييم محددة.
- **عدم التعديل التلقائي:** لا يُطبَّق أي عامل معايرة على `market_value` إلا بقرار صريح من المحلل.
- **قابلية إعادة الإنتاج:** تصدير/استيراد الجلسة يُتيح إعادة إنتاج النتائج الكاملة.
- **شفافية Sandbox:** كل صف في نتيجة Sandbox يحتفظ بـ `original_market_value` للمقارنة.

| لا استمرارية في قاعدة البيانات | البيانات تُفقد عند إغلاق المتصفح | صدِّر الجلسة JSON دائماً بعد كل تشغيل مهم |

## 9. المرحلة التالية الموصى بها — Phase 4: Heritage / Historical Engine

### التبرير

Phase 3 يمنح المنصة عموداً فقرياً قوياً للتقييم الجماعي المعياري. البند التالي على خارطة الطريق الأصلية هو **محرك التراث والعقارات التاريخية**، الذي يُعالج فئةً خاصةً من العقارات ذات خصوصية قانونية وعمرانية وتراثية.

### ما يشمله Phase 4

**النمذجة التخصصية:**
- قيود الاستخدام القانونية على العقارات التراثية (Heritage HBU Restrictions)
- تكلفة الترميم ومعايير الحفاظ (Restoration Cost / Preservation Burden)
- علاوة التراث / خصم السوق (Heritage Premium / Marketability Discount)
- تحليل الاستخدام الأمثل المقيَّد (Restricted HBU Analysis)
- قيود التسويق والإيجار (Marketability Limits)

**مدخلات التقييم الجديدة:**
- تصنيف التراث (UNESCO / وطني / محلي)
- درجة التدهور والحالة الإنشائية
- كلفة الترميم المُقدَّرة
- الفترة التاريخية للإنشاء
- القيود المفروضة قانونياً على الاستخدام والتعديل

**المخرجات المتوقعة:**
- `heritage_adjusted_value` بعد تطبيق كل العوامل التراثية
- `heritage_premium_applied` / `heritage_discount_applied`
- تقرير تقييم تراثي متخصص في XLSX

### الربط مع Phase 3

بيانات Phase 3 (zone_id، property_class، model_version، governance) ستُغذّي مباشرةً محرك Phase 4، مما يُتيح ربط العقارات التراثية بمناطقها السوقية ودورات إعادة التقييم الخاصة بها.
